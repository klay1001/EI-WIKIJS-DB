Overview

本文件記錄系統更新 patch過程的操作，目標系統是 SAP test system(SOEVM221)。採用 SAP Update Manager 2.0。




Reference

 




Pre-requisite

操作前應先準備

l  目標系統 client 000產生 xml檔案

l  SAP Maintenance Planner執行相關程序，產生新的 xml

l  下載由 Maintenance Planner提供的相關patch(包含 SUM安裝程式)

 

 

Procedure

0.      檢查目錄權限，並以 root執行步骤 SUMSTART confighostagent <SID>，在視窗中會提供 url請複製貼於瀏覽器登入 SUM，登入帳號為 <SID>adm

 

1. 完成 Maintenance Planner的相關步驟，並下載正確 XML，填入正確路徑

 

2. DB此次不執行migration，DATABASE TYPE勾選 No migration

 

 

3. 以下步驟可以是否上線來判斷

a.     若系統新安裝未上線時可套用，不會建立 shadow system而是直接進入 downtime 運作

b.    若系統已上線則考慮使用此選項，系統會先建立 shadow system來運作，而讓使用者還能正常作業，等到正式開始要進入 downtime時才會將 shadow system資源匯入，此模式也會較消耗資源，但對使用者影響較低

c.     更優化的運作方式，但是最耗資源，一般實務上也不太執行(無經驗)

 

 

4.Check archive authenticity 及 Switch expert mode on皆不勾選

 

5.提供 Support Package存放目錄，若 SUM在安裝中有缺少 SP，在此步驟會示，請再補上後重新掃描目錄即可

 

6. 輸入 DDIC, DB-Specific 密碼

 

 

7. /usr/sap/T11/SUM/abap/log/CHECKS.TXT可檢視相關檔案解壓縮結果

 

8.Keep archiving on during the whole procedure 不勾選

 

9.此步驟為之後執行時，提供 process數量，若不清楚資源使用是否足夠可直接依照系統提供

10.Missing SAP Notes�.   ( checking: /usr/sap/T11/SUM/abap/log/CHECKS.TXT)

請手動補上需要的 SAP Note

a.download these SAP notes   b.extrace SAR files and ZIP files(only need txt files)  c.upload these txt files using SNOTE

Implement SAP Notes

退回上一步再重新執行

 

11.因 /usr/sap/T11/SUM/abap/data空間不足進行處理

 

12.此次作業並無任何 CR需處理，直接點選 Next

 

13.檢查後選取 Next

 

14.Database Free Space is not enough (CHECKS.TXT)

Message from CHECKS.TXT

!!! Create Tablespace “PSAPSR3750X” for shadow system using BRTOOLS(此例是參考  DBFPLUSD.RES之指令新增 tablespace)

Return SUM and click Next

 

15.系統提醒需要以下 SAP Note

有兩種方式可執行 a. Report: SCWN_TRANSPORT_NOTES  b.停止 SUM後用 SPAM 加入 SAP Note: 2257143

修正後再次進行檢查已正常

 

16.提示接著會鎖定

需要選取 *Lock the development environment now才能選取 Next，並且除了 DDIC, SAP*外其餘帳號暫時無法登入ECC

以下情況如果系統被鎖定，此時除了 SAP*, DDIC外均無法登入，請執行以下指令暫時解除 lock

/usr/sap/T11/SUM/abap/bin/SAPup unlocksys

Release 完成請務必再次鎖住系統 SAPup locksys

 

17.CR release完成後接著進行 Clone tables

上述若遇到 Undo Retention問題，請執行 repeat應可解決 (perhaps resources issue)

 

18.繼續執行後續程序

 

19.以下提示準備進入 System downtime，請依內容說明逐一檢查

 

20.上述檢查完成後，系統接著執行後續步驟

 

21.Execution執行完成，接著 Postprocessing

 

22.此步驟說明downtime已完成，系統會解除 lock情況

 

23.Check transport using SPAU in SAP GUI

SPAU  �.  Check the tab “Without Assistant”

Confirm obsolete SAP Note and needs a Change Request.

24.確認系統描述內容，若無問題再選取 Next

 

25.SUM is OK

 

執行 cleanup之後 /usr/sap 由 59% à 9%，推測此作業大約使用 15GB空間
